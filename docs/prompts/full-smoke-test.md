# Full site smoke test — Chrome extension (thorough)

End-to-end walkthrough of everything shipped on the web recently — 18 parts, roughly 20 minutes if you move at a steady pace. Paste the whole block into Claude Chrome.

**Prereqs before starting:**
- Open `https://sneakersterminal.com` in the same Chrome profile and sign in once with an admin email (or an existing invited account). The extension inherits that session via cookies — without this, Part K + Part M + L will 302 to `/signup` and stall.
- Have two throwaway emails ready (use `+abc`/`+def` aliases on your real inbox if you want to see the confirmation mail, or burner addresses if you don't).
- If the deploy is on a Vercel preview URL (not production), substitute throughout.
- DevTools Network tab open for Part P.

---

You're smoke-testing the Sneakers Terminal web app end-to-end. Work through Parts A–R in order. Each part is a block of checks; mark PASS/FAIL with specifics. Screenshot anything visually broken. Compile the final report at the bottom.

## Part A — Landing page core render

1. Visit `https://sneakersterminal.com/` in an **incognito window** (fresh state, no referral cookie, no session).
2. Check:
   - Hero background image loads (not a blank dark rectangle; you can see the skyline / atmosphere through the dark overlay).
   - Sneakers logo appears in a dark rounded circle with a faint emerald ring + outer glow.
   - Small eyebrow text above the logo: `SNEAKERS TERMINAL / v0.0.1 / PRE-LAUNCH`
   - Tagline below logo (green): `All Prediction market prices in one`
   - Secondary tagline (italic white): short line under it
   - Live waitlist counter: `> N OPERATORS IN QUEUE` (N is a real integer, not `NaN` or `undefined`)
   - Top-right pill link: `🎓 ARE YOU A COLLEGE STUDENT?` (clicking goes to `/students`)
   - Waitlist form present: INDIVIDUAL / BUSINESS toggle, email input, `REQUEST ACCESS` button
   - Global footer renders at the bottom with 4 social-icon placeholders + single-line disclaimer

**Report:** counter value, any visual defects, broken images (list them).

## Part H — Waitlist signup edge cases

Run each on the landing form (you can reset the form by reloading):

1. **Empty submit** — click REQUEST ACCESS with the email field empty. Browser's native required-field popup should appear (no JS error).
2. **Invalid email** — `notanemail` then submit. Native validation should block.
3. **BUSINESS toggle** — click BUSINESS, optional company-name field appears. Fill name `Test Fund LLC`, email `test+biz-{random}@sneakersterminal.com`, submit. Success card should render. A small green note about "Business accounts get priority review + early Business-tier access" should have been visible under the form.
4. **Duplicate email** — submit the SAME email you used in Part G again. Expected: routed to `/login?email=...` (not silently shown "access requested"). Verify URL.
5. **Self-referral (subtle)** — open the referral URL from Part G in the same incognito window → submit with the same email as Part G. Expected: attribution is dropped (referrer = same user). The inviter count should NOT increment. Note the response.

**Report:** all 5 behaviors matched expectations? Flag any that did not.

## Part O — API probes

Hit each endpoint directly in a new tab or via DevTools / `curl` equivalent. Note status and shape.

1. `GET https://sneakersterminal.com/api/markets/opportunities` — expect HTTP 200, JSON body with keys `platforms`, `lastUpdated`, `opportunities` (array), optional `note` field. On current prod, `opportunities` is likely empty with a `note` about "No scraper data available" (known gap).
2. `GET https://sneakersterminal.com/api/waitlist/count` — expect HTTP 200 with a count number.
3. `POST https://sneakersterminal.com/api/waitlist` with body `{"email":"invalid"}` → expect HTTP 400 with `{"error":"invalid_email"}`.
4. `POST https://sneakersterminal.com/api/waitlist` with body `{}` → expect HTTP 400.
5. `GET https://sneakersterminal.com/api/opengraph-image` — should 404 or similar (it's served from the file-convention route at `/opengraph-image`, not `/api/...`). If it 200s unexpectedly, note.

**Report:** each endpoint's status + basic shape check.
